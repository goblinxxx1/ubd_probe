# Ревізія обох фронтів — аудит (звіт)

**Дата:** 2026-07-18. Стан: `main` HEAD `80d4086` (public+admin редизайн + косметика влиті).
Мета аудиту — зібрати знахідки когерентності + a11y + прогалини public-хвостів. **Без змін коду** —
за цим звітом окремо робимо spec→план→фікси.

Контраст рахований за WCAG 2.1 (нормальний текст AA ≥ 4.5:1; великий ≥ 3:1; великий = ≥24px або ≥18.66px bold).
Більшість muted-токенів вживаються на дрібному тексті (12–15px, не bold) → потрібно 4.5.

## B. A11y — не-контраст

- **B1. Немає видимих focus-стилів.** Ніде в public нема `:focus-visible`; кастомні контроли (`.btn`, `.filters__trigger`, `.card__link`, кнопки Pagination) покладаються на дефолт браузера, який на стилізованих кнопках буває слабкий/відсутній. WCAG 2.4.7. → додати помітний `:focus-visible` (2px outline `@dark`/`@link` + offset). Admin: Element Plus має власні focus-стилі — перевірити, чи достатньо.
- **B2. Порожній `alt=""` на фото оффера** (`OfferCard.vue:24`, `OfferDetailView.vue:57`). Маркує декоративним. Захищувано (герой — назва провайдера текстом), але фото репрезентує провайдера → інформативне. **Рішення:** `alt` = назва провайдера, чи лишити декоративним.
- **B3. Статичний nav «Оффери · Про нас»** (`SiteHeader.vue:7`) — текст у `<nav>`, стилізований як клікабельний (uppercase, letter-spacing), але це не посилання й «Про нас» не має роуту. Плюс той самий контраст-фейл. **Рішення:** зробити «Оффери» справжнім router-link і прибрати «Про нас» (YAGNI); або лишити як не-інтерактивний підпис (змінити вигляд).
- **B4. Дрібне:** filters-тригер без `aria-expanded`/`aria-controls` (disclosure); Pagination `<nav>` без `aria-label`. Легко додати.

## C. Когерентність / public-хвости

- **C1. Колір помилки поза брендом.** `@danger #b00020` (холодний кармін) не з палітри; бренд-теракота = **#C0492B** (4.75 AA). Вживається у OfferGrid error + admin-alias. → замінити на теракоту.
- **C2. Pagination простенька.** `.btn` майже-біла, radius 8 (картки 12), без hover/акценту (лише opacity на disabled). `OfferGrid`-стани — голий центрований `<p>`. → узгодити: hover (`@cream` bg + `@brand`/`@link` рамка), стани у делікатній панелі (cream + рамка) замість голого тексту.
- **C3. `OffersView` h1** «Знижки та події для УБД» 24px успадковує вагу body (400) — слабка ієрархія проти карток (900). → підняти вагу заголовка сторінки (700–900).
- **C4. Хардкод radius 8px** у `.btn`/filters/pagination/admin OfferFilters — не токен. → ввести `@radius-sm: 8px` (DRY), і дзеркалити в admin.
- **C5. Паритет токенів public↔admin — ОК.** Admin вірно дзеркалить спільні токени; розбіжностей палітри нема. (Спостереження з косметики: admin table header виграє лише за порядком джерел — робастніше через `--el-table-header-bg-color`, але працює.)

## Пропоновані батчі фіксів (для spec→плану)

1. **Контраст-токени** (A1–A4): затемнити `@nav-muted`, `@meta-muted`(=desc), `@placeholder`; амбер-як-текст → `@link` (public `a` + admin active-nav). Дзеркалити зміни public↔admin. Візуальна зміна — легша/темніша muted-типографіка, посилання стають темно-бурштиновими.
2. **Focus-стилі** (B1) — `:focus-visible` на всі кастомні контроли обох фронтів.
3. **Public-хвости** (C1–C4): теракота-помилка, рестайл Pagination (hover/акцент), стани OfferGrid у панелі, вага h1, токен `@radius-sm`.
4. **Дрібний a11y** (B2 alt, B3 static-nav, B4 aria) — за рішеннями нижче.

## Рішення, потрібні перед spec

- **D1.** Header «УБД» амбер на білому: лишити як логотип-акцент чи затемнити до `@link`?
- **D2.** Фото оффера `alt`: назва провайдера (інформативне) чи лишити декоративним `alt=""`?
- **D3.** Static nav «Оффери · Про нас»: зробити «Оффери» справжнім лінком + прибрати «Про нас»; чи лишити не-інтерактивним підписом?
- **D4.** Обсяг: робимо всі 4 батчі, чи розбиваємо/відкидаємо якийсь?
