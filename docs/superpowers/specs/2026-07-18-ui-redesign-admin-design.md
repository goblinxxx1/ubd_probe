# UI-редизайн адмінки — дизайн (spec)

**Дата:** 2026-07-18
**Гілка:** `feat/ui-redesign-admin` (від `main`, HEAD `019addc`)
**Попередній трек:** public-редизайн (влитий у `main`) — див. `docs/superpowers/specs/2026-07-16-ui-redesign-public-design.md`

## Мета

Оновити вигляд адмінки (`admin/`, Vue3/JS/Element Plus) у тому ж світлому бурштиновому
стилі, що й публічний сайт, **максимально просто** — не навантажуючи адмінку кастомними
стилями. Адмінка — щільний робочий інструмент (таблиці, форми), тож читабельність
пріоритетна над декоративністю.

Поточний стан адмінки: майже голий Element Plus. `variables.less` = `@brand: #1f6feb`
(синій) + ширина сайдбару; `main.js` тягне дефолтний `element-plus/dist/index.css`; у
`AdminLayout` захардкоджені синьо-сірі кольори (`#f5f7fa` сайдбар); шрифт `system-ui`.

## Принципові рішення (узгоджені на брейнстормі)

1. **Механізм теми:** override CSS-змінних `--el-*` у глобальному стилі. Element Plus 2.x
   будує компоненти з CSS custom properties; перевизначення `--el-color-primary` + відтінків
   і кількох нейтралей у `:root` автоматично перефарбовує кнопки, теги, активні стани,
   пагінацію, фокус-рамки форм — **без змін білд-конфігу, без SCSS, без per-component хаків**.
2. **Шрифт UAF Memory — лише заголовки/бренд.** Логотип, `h1`/`h2` заголовки сторінок,
   назви розділів у сайдбарі. Тіло таблиць/форм/інпутів лишається `system-ui` (щільні дані —
   максимальна читабельність).
3. **Статуси лишаються семантичними** кольорами Element Plus (success/warning/danger/info),
   НЕ брендуються в бурштин — найкраща скануваність статусів оферів.
4. **Клікабельні посилання** (єдине функціональне доповнення): у таблиці оферів/черзі
   модерації та в таблицях джерел.
5. **Таблиці/пагінація/теги — дефолтні**, лише перефарбовані через тему + легкі штрихи.

## Скоуп

Усі 8 view охоплюються **автоматично** через тему (Offers, Moderation, Sources,
SuggestedSources, Categories, Users, Login, OfferForm). Ручні правки — лише:

- **A. Токени** — розширити `admin/src/styles/variables.less`.
- **B. Шрифт** — `admin/src/styles/fonts.less` + woff2-асети.
- **C. Хрест public `font-weight:800`** — нормалізація в `public/` (прекондиція консистентності).
- **D. Chrome** — `AdminLayout.vue`, `LoginView.vue`.
- **E. Клікабельні посилання** — таблиці оферів/модерації + джерел.
- **F. Легкі штрихи таблиць** — фон шапки, hover рядка (через тему/глобальний стиль).

Поза скоупом: жодних змін backend/API/crawler; жодного нового рефакторингу логіки view;
жодних нових полів моделі.

## Деталі за секціями

### A. Токени (`admin/src/styles/variables.less`)

Дзеркало значень public (єдине джерело істини кольорів — ті самі, що у
`public/src/styles/variables.less`):

```less
// UI-редизайн адмінки — світлий мінімалізм, бурштиновий акцент (дзеркало public-токенів)
@bg: #FAFAF9;            // тло контенту
@surface: #FFFFFF;       // сайдбар/топбар/картки
@text: #14110A;          // основний текст
@brand: #E0982A;         // бурштин (primary)
@divider: #E7E5E0;       // тонкі лінії
@nav-muted: #8A8578;     // неактивна навігація
@meta-muted: #7E7768;    // лейбли, мета
@cream: #F5F1E8;         // крем-панель (login-картка, активний пункт)
@dark: #211D16;          // темні акценти
@heading-weight: 900;    // вага заголовків UAF Memory (консистентно з public)
@sidebar-width: 220px;   // (зберігається)
```

`@brand` більше НЕ `#1f6feb`. Старе значення прибрати.

### B. Шрифт (`admin/src/styles/fonts.less` + `admin/src/assets/fonts/`)

- Скопіювати 5 файлів `UAFMemory-*.woff2` з `public/src/assets/fonts/` у
  `admin/src/assets/fonts/`.
- Створити `admin/src/styles/fonts.less` — дзеркало `public/src/styles/fonts.less`
  (`@font-face` для ваг 300/400/500/700/900).
- Імпортувати `fonts.less` у `global.less`.
- Застосувати UAF Memory **лише** до: логотип сайдбару, `h1`/`h2` (заголовки view),
  назви пунктів навігації сайдбару. `body` лишається `system-ui`.
- Вага заголовків = `@heading-weight` (900). Жодних `font-weight:800` в адмінці.

### C. Хрест public `font-weight:800` → `900`

Нормалізувати 4 місця в `public/` (нуль візуальних змін — зараз 800 і так рендериться як
900 Black, бо фейсу 800 немає):

- `public/src/views/OfferDetailView.vue:93` (`.detail__provider`)
- `public/src/components/SiteHeader.vue:16` (`.brand`)
- `public/src/components/OfferCard.vue:67` (провайдер картки)
- `public/src/components/OfferBadge.vue:18` (бейдж)

Це закриває відкладений follow-up і робить токен ваги заголовка чесним на обох фронтах.
Public-тести мають лишитися зеленими (стилі, не логіка).

### D. Chrome (`AdminLayout.vue`, `LoginView.vue`)

**AdminLayout:**
- Сайдбар: `background: @surface` (білий), тонкий правий дільник `1px @divider`
  замість синьо-сірого `#f5f7fa`. Прибрати захардкоджені кольори. (`@cream` — лише для
  підсвітки активного пункту.)
- Логотип «UBD»: UAF Memory, бурштин `@brand`, вага заголовка.
- Пункти навігації: неактивні `@nav-muted`; активний (`router-link-active`) — бурштинова
  індикація (текст `@brand` + лівий бордер/підсвітка `@cream`).
- Топбар: `background: @surface`, нижній дільник `1px @divider`. Роль/кнопка «Вийти»
  лишаються, кнопка успадковує бурштинову тему.
- Бейдж лічильника модерації (`el-badge`) — лишається (семантичний).

**LoginView:**
- Тло `@bg`, центрована крем-картка (`@cream`, тонкий бордер `@divider`, radius, без
  важких тіней). Заголовок UAF Memory. Кнопка входу — бурштинова (через тему).

### E. Клікабельні посилання

**Таблиця оферів / черга модерації** (`OffersListView.vue`):
- Додати нову колонку «Джерело», що рендерить `site_url` як `<el-link>`
  з `type="primary"`, `target="_blank"`, `rel="noopener noreferrer"`. Для оферів типу
  «новина» показувати `article_url`, якщо є. Порожній URL → нічого (без битого лінка).
- Мета: модератор відкриває джерело й перевіряє офер перед публікацією.

**Джерела** (`SourcesView.vue`, `SuggestedSourcesView.vue`):
- `url_or_handle` для рядків типу `website` — `<el-link>` з `target="_blank"`,
  `rel="noopener noreferrer"`. Для соц-handle (не валідний URL) лишається текст.

Функціональна вимога: посилання відкриваються в новій вкладці, з `rel="noopener"`.
Кожна зміна покривається фокусним тестом (рендер лінка для website / текст для handle;
рендер лінка офера за наявності site_url).

### F. Легкі штрихи таблиць

Через тему/глобальний стиль (не per-view): фон шапки `el-table`, hover рядка в теплих
нейтралях. Без важких кастомних override-ів — «максимально просто».

## Тестування

- Vitest-набір адмінки має лишитися зеленим (наразі 69 тестів).
- Нові фокусні тести на клікабельні посилання (E).
- Public-набір зелений після нормалізації 800→900 (C).
- Візуальна перевірка в браузері (тимчасова preview або dev-сервер): сайдбар/топбар/логін/
  таблиці/форми — світлий бурштиновий стиль, читабельні дані, робочі посилання.

## Ризики / нотатки

- Element Plus версія має підтримувати `--el-color-primary` CSS-vars (2.x — так). Перевірити
  фактичну версію в `admin/package.json` на етапі плану.
- Перефарбування primary впливає на ВСІ компоненти — переконатися, що семантичні статуси
  (success/warning/danger) не зачеплені (вони окремі `--el-color-*`).
- Копія woff2 в адмінку дублює асети public (прийнятно — фронти незалежні, як домовлено).
