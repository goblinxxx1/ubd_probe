# UI-редизайн — публічний сайт (Track public)

**Дата:** 2026-07-16
**Гілка:** `feat/ui-redesign` (від `main`)
**Скоуп:** тільки публічний фронтенд (`public/`). Адмінка — окремий наступний трек зі своїм spec/планом. Бекенд і crawler **не змінюються**.

## Мета

Оновити вигляд публічного сайту зі старої синьо-білої палітри на світлий мінімалізм із бурштиновим акцентом та шрифтом UAF Memory. Дизайн-напрям узгоджено на візуальному брейнстормі 2026-07-16 (мокап: `.superpowers/brainstorm/1991-1784226233/content/public-page-light.html`).

## Ключова знахідка (звужує скоуп)

Поле `description` **уже існує** в моделі оффера (`backend/app/models/offer.py`, `Text`), у схемі `OfferOut` і віддається публічним API. Тобто блок «опис компанії» на картці рендериться з `offer.description` без жодних змін бекенду/crawler. Це виправляє застарілий запис у памʼяті, який стверджував, що поля нема.

## Мапінг полів оффера на новий дизайн

Семантична інверсія відносно поточної картки (де `title` — головний заголовок, `provider` — приглушений підпис). Підтверджено користувачем.

| Елемент | Поле |
|---|---|
| Герой (велика назва бізнесу) | `provider` |
| Бейдж (`−20%` / «Безкоштовно» / «Подія») | `discount_type` + `discount_value` + `type` (через `offerBadge()`) |
| Короткий текст біля бейджа | `title` |
| Блок опису | `description` (курсивний плейсхолдер `[опис]`, якщо порожньо) |
| Чипи «Для кого» | `target_categories` |
| Футер-мета | `offer_categories` · `location` |
| Футер-посилання | `links` / `site_url` / `article_url` |

## Компоненти

### `OfferCard.vue` (переструктурування)
Зображення з картки **прибираємо** (текстовий мінімалізм). Структура згори вниз:
1. **`provider`-герой** — bold, до 2 рядків, `line-height:.95`, `font-size` ~24px, `letter-spacing:-.3px`. Клікабельний → сторінка-деталь.
2. **Рядок знижки** — `<OfferBadge>` + `title` (короткий текст, напр. «на все меню»).
3. **Блок опису** — `offer.description`; якщо порожній — курсивний `[опис]` кольором placeholder.
4. **Панель «Для кого»** — лейбл «ДЛЯ КОГО» (uppercase) + чипи `target_categories`. Панель **ховається**, якщо `target_categories` порожній.
5. **Футер** — верхня лінія-розділювач; ліворуч `offer_categories · location` (uppercase meta), праворуч джерела-посилання (`links`, з фолбеком на `site_url`/`article_url`; кілька джерел нумеруються — поведінка як зараз). Зовнішні посилання `target="_blank" rel="noopener"`.

Навігація: клік по картці/герою відкриває маршрут `offer` (деталь); клік по посиланнях джерел веде на зовнішній сайт (не тригерить навігацію на деталь).

### `OfferBadge.vue` (тільки стилі)
Логіку `offerBadge()` в `utils/format.js` (текст/kind) **не чіпаємо**. Три види, текст **вертикально по центру** (flex `align-items:center`):
- `discount`: bg `#E0982A`, текст `#221806`
- `free`: bg `#DDE7D5` (шавлієвий світло-зелений), текст `#221806`
- `event`: bg `#211D16`, текст `#E0982A`

### `SiteHeader.vue`
Білий фон, бренд «Знижки для **УБД**» (слово «УБД» бурштином `#E0982A`), навігація праворуч (uppercase, `letter-spacing`, `#8A8578`), нижня лінія-розділювач `#E7E5E0`. Прибрати темні плашки старого дизайну.

### `OfferFilters.vue`
Фільтри-пігулки: неактивна — світла з рамкою `#E3E1DB`; активна — темна `#211D16` / текст `#F3EFE4`. Зберегти поточну поведінку (джерело істини — URL, dropdown-панель).

### `SiteFooter.vue`
Світлий мінімалізм у новій палітрі (розділювач + приглушений текст). Без темних плашок.

### `OfferDetailView.vue`
Ті самі токени, просторіша версія: більший герой-`provider` (до ~38px), повний `description`, рядок `бейдж + title`, панель «Для кого», категорії, локація, **усі** джерела-посилання, дати чинності (`valid_from`/`valid_until`, форматувати наявним `formatDate`). Фото-плейсхолдер не показуємо; якщо є `image_url` — рендеримо як логотип/візуал.

### `App.vue` / глобальні стилі
Оновити глобальний фон `#FAFAF9`, базовий колір тексту, підключення шрифту (перевірити, що UAF Memory застосовано глобально).

## Тестування

Розмітка й класи змінюються → оновити компонентні тести під нову структуру, зберігаючи перевірки **поведінки** (а не конкретних старих класів):
- `OfferCard` — рендер провайдера-героя, опису/плейсхолдера, чипів «для кого», джерел-посилань (одне й кілька), навігація на деталь.
- `OfferBadge` — тексти («−20%», «Безкоштовно», «Подія») і клас-модифікатор виду.
- `OfferFilters`, `SiteHeader`, `SiteFooter`, `OfferDetailView` — оновити селектори під нову розмітку.

Ціль: зелений Vitest (наразі 43 passed), без регресій поведінки. Візуальна перевірка в браузері (Vite :5174) на реальному/замоканому лістингу та деталі.

## Поза скоупом (свідомо)

- Бекенд / crawler — не потрібні (`description` вже є).
- Адмінка — наступний окремий трек (Element Plus theme override на бурштин + UAF Memory + світлий мінімалізм).
- Ревізія обох фронтів — після реалізації обох треків.
